## 项目简介

Student Manager 是一个用 C 语言实现的学生成绩管理系统，基于单向链表（singly‐linked list）设计。尽管代码量不足 500 行，但被拆分成多个模块，从而保持低耦合、高内聚。该项目适合作为教学演示用、练习用，也可作为基础框架进行二次扩展。

## 主要特性

- 使用通用链表模块（`list`）处理通用节点逻辑，无业务耦合。
- 使用学生业务模块（`student`）定义学生结构体 + 特定业务规则，仅依赖链表接口。
- 使用终端用户界面模块（`ui`）实现与用户交互，仅调用 `list` 与 `student` 的公开接口。
- `main` 模块仅负责启动 UI，最高层高度抽象。
- 提供“学生成绩”的 **增加、删除、查询、排序** 功能。
- 模块化设计便于替换或扩展：例如将链表替换为动态数组、将终端界面替换为 GUI 或 Web 界面、将“学生”替换为“图书”“商品”等，无需修改底层链表模块。

## 模块结构

| 模块      | 职责                      | 依赖               | 备注                                 |
| --------- | ------------------------- | ------------------ | ------------------------------------ |
| `list`    | 通用链表（`void*` 节点）  | 无业务依赖         | 可复用于任何类型数据                 |
| `student` | 学生结构体定义 + 业务规则 | `list`             | 定义学生属性（例如姓名、学号、成绩） |
| `ui`      | 终端交互界面              | `list` + `student` | 实现菜单、输入、输出、调用业务函数   |
| `main`    | 程序入口                  | `ui`               | 简洁启动流程，不涉及业务细节         |

## 快速开始

### 1. 克隆仓库

```
git clone https://github.com/Sensensen666/First_27.git
cd First_27
```

### 2. 编译程序

你可以采用以下两种主要方式：

**方式 A：使用 CMake（推荐）**

```
mkdir build
cd build
cmake ..
cmake --build .
```

**方式 B：裸 gcc 编译**（假设所有 `.c` 文件在 `src/`，头文件在 `include/`）

```
gcc src/*.c -I include -o student_mgr
```

生成的可执行文件（例如）为 `student_mgr`（在 Windows 上可能是 `student_mgr.exe`）。

### 3. 运行程序

```
./student_mgr
```

然后根据屏幕提示，使用菜单操作学生成绩系统。

## 用法说明

- 启动后会展示主菜单，例如 “添加学生”、“删除学生”、“查询学生”、“排序学生成绩”、“退出” 等选项。
- **添加学生**：输入学生姓名、学号、成绩等字段。
- **删除学生**：按学号或其他关键字段删除。
- **查询学生**：输入学号或姓名查找并显示学生信息。
- **排序学生成绩**：按成绩高低或其他规则排序链表，并展示结果。
- 所有操作通过链表模块实现增、删、查、遍历排序，UI 模块只负责用户交互。

## 代码组织 & 设计理念

- **低耦合设计**：链表模块与业务数据（学生）模块解耦，链表只是处理通用节点与指针操作。
- **模块化设计**：业务模块与 UI 模块分离，可替换 UI（例如 GUI 或 Web）或者替换数据类型（例如从“学生”换为“图书”）而不需改动链表逻辑。
- **易于扩展与维护**：例如5 分钟内可将链表改为动态数组，10 分钟内可将终端换为 Qt/WebAssembly，15 分钟内可将“学生”换成“图书”“商品”——无需改动底层 list 模块。
- **教学友好**：整体代码量较少，仅数百行但是结构清晰，非常适合用于讲解数据结构 + 模块化设计 +实际案例。

## 扩展建议

- 增加对 “学生”更丰富字段（如性别、专业、年级、联系电话）。
- 支持从/导出学生数据至文件（如 CSV 或二进制格式）。
- 支持更多排序规则（如按照年级、姓名、成绩升降序）。
- 使用图形界面（例如 GUI）或 Web 界面替代终端。
- 支持数据持久化（例如 SQLite、JSON 存储）。
- 将链表替换为其他数据结构（如动态数组、双向链表、平衡树等）。
- 集成单元测试框架以自动测试链表模块与业务模块。

## 依赖 & 构建环境

- 编译器： gcc 或 clang，支持 C99 或更高版本。
- 可选：CMake 用于生成构建系统。
- 操作系统：任何支持标准 C 的系统（Linux、macOS、Windows+MinGW/Cygwin 均可）。
- 无其他第三方库依赖。
